## Pull requests
- NEVER create PRs against upstream repos. This is a fork (xPathin/GWToolboxpp).
- Always use `--repo xPathin/GWToolboxpp` or `-R xPathin/GWToolboxpp` with `gh pr create`
- `gh` defaults to the parent/upstream repo when working from a fork. You MUST override this.

## Merging PRs
- Prefer squash merge
- Use merge only when the commit history gives real extra value
- Never rebase

## Commit messages
- Concise, one line, tag pattern (fix:, feat:, chore:, etc.)
- No Co-Authored-By lines
- No AI attribution
